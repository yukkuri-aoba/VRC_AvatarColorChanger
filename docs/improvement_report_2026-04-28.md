## 2. 主要改善①：AA 境界の halo 除去（Color Decontamination）

### 問題

AA（アンチエイリアス）境界ピクセルは元テクスチャ生成時に次式で合成されている：

```
pixel_AA = α × FG + (1-α) × BG   （α ∈ (0,1)）
```

従来アルゴリズムはこの混色に対して HSV 変換を直接適用していたため、「彩度が低くくすんだ中間色（halo）」が境界に残っていた。

**具体例（Feina/Clothes.png、バンダナ→青変換）:**

| x 座標 | 元ピクセル | 変換後（変更前） | 変換後（変更後） |
|---|---|---|---|
| 3097 | (247,237,227) | (239,234,232) | (247,237,227) ← 変化なし（BG正常） |
| 3098 | (248,239,229) | (232,234,237) ← **青寄りグレー** | (248,239,229) ← 変化なし |
| 3099 | (242,221,210) | (203,213,233) ← **薄青** | (210,224,248) ← 正しいα合成 |
| 3100 | (230,138,119) | (62,120,218)  | (62,120,218)  ← 正しい青 |

### 解決策

α 分解 + 再合成（alpha matting）：

```
BG 推定：  近傍 R px 内の strength=0 ピクセルの平均色
α 推定：   clip( (pixel - BG) · (sample - BG) / ‖sample - BG‖², 0, 1 )
再合成：   result = α × target + (1-α) × BG
```

### 実装

- `DecontaminateAaBoundary()` — α 分解・再合成のコア処理  
- `BoxFilterSum()` — O(N) 可分ボックスフィルター（水平→垂直パス）で BG 推定を高速化

### 適用条件

| strength 値 | 処理 |
|---|---|
| = 0 | 変化なし（背景） |
| 0 < s < 0.97 | **decontamination**（AA 境界） |
| ≥ 0.97 | HSV transfer（内部、陰影保持のため従来通り） |

### 新 UI パラメータ

- **境界クリーンアップ（α分解）** トグル — デフォルト ON  
- **α分解 近傍半径** スライダー（アドバンスドモード）— デフォルト 4px、範囲 1〜12

## 3. 主要改善②：低彩度サンプルの色距離計算

### 問題

サンプル色の彩度 S が低い場合（例：白、グレー、クリーム）、HSV の Hue（色相）はノイズに近くなる。従来の HSV 距離式はこの状況で隣接する低彩度の別素材（クリーム色の服など）と誤マッチしやすかった。

### 解決策

**RGB-HSV ハイブリッド距離：**

```
chromaConfidence = clamp01( (sS - 0.05) / (0.15 - 0.05) )
dist = lerp( rgbDist, hsvDist, chromaConfidence )
```

- `sS < 0.05`：RGB ユークリッド距離のみ使用（色相を完全無視）
- `sS > 0.15`：従来通り HSV 距離
- その間：線形ブレンド

**ハイライト補助の色相キャップ：**

ハイライト補助分岐（鏡面反射）の色相許容幅を `Max(0.05, tolerance × 0.3)` に制限。クリーム色などの低彩度隣接色がハイライトとして誤マッチするのを防ぐ。

## 4. 主要改善③：ゾーン別除外マスク

### 変更前

除外マスクは全ゾーン共通の 1 枚のみ。

### 変更後

- **共通マスク**（全ゾーンに適用）と**ゾーン別マスク**（特定ゾーンのみ）を個別管理
- 処理時は OR 結合して適用
- ゾーン識別用 GUID（`ColorZone.id`）で永続管理
- プリセット保存/読み込みにマスクを同梱可能（ON/OFF 切替）
- プレビューに各ゾーンのマスクオーバーレイを HSV 黄金比色で描画
